# 硬话软说 MVP 网站设计规格

## 1. Concept & Vision

**产品定位**：职场吐槽翻译官 —— 把心里话翻译成体面话

**核心体验**：用户输入对领导/客户/同事的吐槽，AI 瞬间转换成高情商、得体、专业的对外回复。不是写作优化工具，而是「职场情绪翻译器」——输入是情绪，输出是智慧。

**体验基调**：有趣、有温度、像懂你的职场老江湖。不是冷冰冰的 AI，而是你的「嘴替」。

**参考风格**：we0.ai 的卡通人物 + 手绘质感 + 留白呼吸感，但配色更温暖活泼，适合「吐槽→体面」这个反差感场景。

## 2. Design Language

### 色彩系统
```
背景主色:     #FFF9F5 (暖白，像纸张的温柔感)
背景次色:     #FFF0E8 (浅蜜桃，用于卡片和区块)
品牌主色:     #FF6B35 (活力橙，代表情绪出口)
品牌次色:     #4ECDC4 (薄荷绿，代表体面得体)
品牌强调:     #FFE66D (明亮黄，用于高亮和按钮悬停)
文字主色:     #2D3436 (深灰，柔和但不阴沉)
文字次色:     #636E72 (中灰，用于副文本)
吐槽区背景:   #FFF5F5 (淡粉红，暗示「这里在说情绪」)
回复区背景:   #F0FFF4 (淡薄荷，暗示「这里是体面话」)
边框色:       #E0E0E0
阴影色:       rgba(255, 107, 53, 0.1)
```

### 字体
- **标题**: "ZCOOL KuaiLe" (站酷快乐体) + "Noto Sans SC" fallback — 可爱有活力
- **正文**: "Noto Sans SC" 400/500 — 清晰易读
- **Slogan/标签**: "ZCOOL KuaiLe" — 增加趣味性

### 动效哲学
- **入场动画**: 元素从下方 fade+slide 入场，stagger 100ms，有弹跳感 (cubic-bezier(0.34, 1.56, 0.64, 1))
- **转换按钮**: 点击时缩放 0.95 + 震动反馈，模拟"翻译中"的期待感
- **结果展示**: 左侧吐槽区和右侧回复区用不同颜色区分，切换时有滑动翻转效果
- **悬停效果**: 卡片轻微上浮 (translateY -4px) + 阴影加深
- **打字机效果**: AI 回复结果逐字显示，模拟真人在打字
- **粒子/烟花**: 首次转换成功时轻微的庆祝粒子动画（不是大场面，是点小惊喜）

### 图标 & 插图风格
- **人物插图**: 3-4个卡通职场人物（参考 we0.ai 的手绘风格但更圆润可爱）
  - 一个"正在吐槽的打工人"（有点丧但可爱）
  - 一个"高情商 AI 化身"（温和微笑的机器人或职场老鸟）
  - 一个"被翻译结果打动的人物"（惊讶+满意的表情）
- **场景图标**: 领导 👔 / 客户 💼 / 同事 🤝 / 老板 👑 — 用 Emoji 风格但统一线条粗细
- **装饰元素**: 手绘风格的线条、箭头、对话气泡、星星 ✦ 标记

```
[导航栏] — 极简，左 logo 右一个「开始翻译」按钮

```

### Hero 区域详细设计
- **左半边**: 
  - 主标题: "硬话软说" (超大字号，ZCOOL KuaiLe 字体，品牌橙色)
  - 副标题: "心里随便吐槽，对外体面发言"
  - 一句话描述: "输入你的职场怨气，AI 把它们翻译成得体、专业、让人舒服的回复"
  - CTA 按钮: 「开始翻译 →」 (品牌橙背景，白字，有微动画)
  - 小标签: "免费使用 · 无需注册 · 5秒完成"
  
- **右半边**:
  - 卡通人物插图 (3个人物从左到右: 吐槽员工 → AI翻译 → 满意结果)
  - 手绘风格的装饰线条和星星
  - 可以有一个"对话气泡"叠加在人物旁边，显示示例翻译

### 响应式策略
- **桌面 (≥1024px)**: 左右分栏 Hero，3列功能卡片
- **平板 (768-1023px)**: Hero 上下排列，功能卡片2列
- **手机 (<768px)**: 全栈布局，卡片单列，演示区精简

## 4. Features & Interactions

### 核心功能：实时演示翻译器

**场景选择器** (v1 MVP 只需5个场景)
- 领导场景: "领导布置的任务不合理/加班太多/功劳被抢"
- 客户场景: "客户反复改需求/预算砍一半还催交期"
- 同事场景: "同事甩锅/总是蹭资源/背后说坏话"
- 晋升/加薪: "年终评定不公平/晋升被拒"
- 离职/交接: "想辞职不知怎么开口/交接时撕破脸"

**交互流程**
1. 用户点击场景卡片（卡片有选中动画，选中后边框变橙色）
2. 输入吐槽内容（textarea，有字数计数，最大300字）
3. 点击「翻译」按钮
4. 加载动画（1-2秒，有微动画，不是无聊的 spinner）
5. 结果展示：左边原始吐槽（淡红色背景），右边翻译结果（薄荷绿背景）
6. 用户可选择：复制结果 / 重新翻译 / 精修（v2功能）

**示例对话**
```
场景: 领导场景
吐槽输入: "领导天天加班到10点，周末也不放过，还觉得理所应当，真把自己当狼性文化了"

AI回复: "我理解你希望工作和生活能够平衡。其实我一直很佩服你对工作的投入，也希望能跟上这个节奏。不过最近连续的高强度工作让我有点吃力，能不能我们找个时间聊聊，看能不能调整一下工作方式，在保证质量的前提下，让我也能有一些休息和充电的时间？"
```

### 功能特色卡片
1. **3秒翻译**: 输入吐槽，选择场景，一键生成体面回复
2. **场景全覆盖**: 领导/客户/同事/晋升/离职，职场全场景
3. **精修打磨**: 对结果不满意？一键重新生成，或手动调整

### 社交货币展示
- 展示 3-5 个"神翻译"截图/案例
- 每个案例包含：吐槽内容（打码）+ 翻译结果 + 用户反馈标签（如"加薪成功🎉"）
- 用卡片轮播或 masonry 布局

## 5. Component Inventory

### 导航栏 (Navbar)
- 左: Logo (文字 logo: "硬话软说")
- 右: 「开始翻译」按钮
- 滚动时: 添加底部阴影

### 场景卡片 (SceneCard)
- 默认: 白色背景，灰色边框，卡通 Emoji + 场景名
- 悬停: 上浮 + 橙色边框 + 阴影
- 选中: 橙色背景(淡) + 橙色边框 + 左侧有选中条
- 点击: 缩放 0.95 反馈

### 输入区 (ComplaintInput)
- Textarea，placeholder: "比如：领导总让我加班到很晚，还觉得理所应当..."
- 有字数计数 (0/300)
- 输入时边框变橙色

### 翻译按钮 (TranslateButton)
- 品牌橙背景，白字
- 悬停: 变亮 + 轻微放大
- 点击: 缩放 0.95 + 震动
- 加载中: 显示"翻译中..." + 动画
- 禁用: 灰色，不可点击（输入为空时）

### 结果展示区 (ResultDisplay)
- 左侧卡片(吐槽): 淡红色背景，显示原始输入
- 右侧卡片(回复): 薄荷绿背景，AI 回复内容，打字机效果入场
- 中间箭头: 手绘风格的箭头符号
- 底部操作: 复制按钮 / 重新翻译按钮

### 功能卡片 (FeatureCard)
- 白色背景，左侧大 Emoji 图标
- 标题 + 描述文字
- 悬停上浮

### 案例展示卡 (TestimonialCard)
- 用户吐槽（局部打码）+ AI 回复 + 效果标签
- 手绘风格的引号装饰

### CTA 按钮 (CTASection)
- 品牌橙大按钮 + 白色文字
- 背景可以有渐变或手绘纹理

## 6. Technical Approach

### 技术栈
- **前端**: 纯 HTML + CSS + Vanilla JavaScript（单文件，便于快速部署）
- **AI 调用**: 直接前端调用硅基流动/智谱 AI API（JSONP 或 CORS 代理）
- **部署**: GitHub Pages / Vercel / Netlify（免费）

### API 设计
```
POST /v1/chat/completions (OpenAI 兼容格式)
{
  "model": "Qwen/Qwen2.5-7B-Instruct",
  "messages": [
    {"role": "system", "content": "你是一个职场高情商回复生成器..."},
    {"role": "user", "content": "场景: 领导场景\n吐槽: {user_input}"}
  ],
  "temperature": 0.8,
  "max_tokens": 500
}
```

### 文件结构
```
ying-hua-ruan-shuo/
├── index.html      (单文件 MVP，包含所有 HTML/CSS/JS)
├── SPEC.md         (本规格文档)
└── README.md       (项目说明)
```

### 质量标准
- Lighthouse Performance ≥ 90
- 移动端完全可用
- 无 console error
- 所有按钮有 click handler
- 加载状态有反馈
